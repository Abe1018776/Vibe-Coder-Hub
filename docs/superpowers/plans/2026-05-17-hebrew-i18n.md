# Hebrew i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Hebrew translation to all public-facing pages of Vibe Coder Hub with a cookie-driven locale switcher, browser-locale default, and full RTL layout when Hebrew is active. Admin dashboard stays English.

**Architecture:** `next-intl` in "no i18n routing" mode. Server-side locale resolution (cookie → `Accept-Language` → fallback) runs in the root layout. Translation messages live in `messages/en.json` + `messages/he.json`. RTL is toggled via `<html dir>` and the public-page Tailwind utilities are migrated from physical (`ml-*`) to logical (`ms-*`) so they auto-flip.

**Tech Stack:** Next.js 15 App Router, `next-intl` 3.x, `@clerk/localizations`, Tailwind v4 logical utilities, Clerk auth, Drizzle, TypeScript strict.

**Verification grain:** This project has no test runner (no `pnpm test` script, no Jest/Vitest). Mechanical verification per task = `pnpm typecheck` + `pnpm lint`. Functional verification = manual smoke against `pnpm dev` (the matrix is in the spec). Do NOT add a test runner — that's scope creep. The `resolveLocale` helper is small and pure enough that the smoke matrix exercises it adequately.

**Commit cadence:** One commit per task (or per logical sub-step where called out). All commit messages follow the existing project style (short imperative subject, no body unless useful — see `git log` for examples like "Add Edit button to Showcase projects for owners").

- [ ] **Step 2: Create src/i18n/request.ts**

Create `/root/Vibe-Coder-Hub/src/i18n/request.ts` with:

```ts
import { getRequestConfig, type IntlErrorCode } from "next-intl/server";
import { resolveLocale, DEFAULT_LOCALE } from "@/lib/i18n/locale";

export default getRequestConfig(async () => {
  const locale = await resolveLocale();

  const messages = (await import(`../../messages/${locale}.json`)).default;
  // Always load English as the prod fallback for any missing keys.
  const fallback =
    locale === DEFAULT_LOCALE
      ? messages
      : (await import(`../../messages/${DEFAULT_LOCALE}.json`)).default;

  return {
    locale,
    messages,
    onError(error: { code: IntlErrorCode; message?: string }) {
      // Dev: log loudly. Prod: stay silent — getMessageFallback handles UX.
      if (process.env.NODE_ENV !== "production") {
        console.warn(`[i18n] ${error.code}: ${error.message ?? ""}`);
      }
    },
    getMessageFallback({ key, namespace }: { key: string; namespace?: string }) {
      // Walk the en.json fallback by "namespace.key" path; if found, return it.
      const path = namespace ? `${namespace}.${key}` : key;
      const value = path
        .split(".")
        .reduce<unknown>(
          (acc, seg) =>
            acc && typeof acc === "object" && seg in (acc as Record<string, unknown>)
              ? (acc as Record<string, unknown>)[seg]
              : undefined,
          fallback,
        );
      return typeof value === "string" ? value : path;
    },
  };
});
```

## Task 4: Create locale resolver

**Files:**
- Create: `src/lib/i18n/locale.ts`

- [ ] **Step 1: Implement resolveLocale**

Create `/root/Vibe-Coder-Hub/src/lib/i18n/locale.ts` with:

```ts
import { cookies, headers } from "next/headers";

export type Locale = "en" | "he";
export const SUPPORTED_LOCALES: readonly Locale[] = ["en", "he"] as const;
export const DEFAULT_LOCALE: Locale = "en";
export const LOCALE_COOKIE = "NEXT_LOCALE";

export async function resolveLocale(): Promise<Locale> {
  const cookieVal = (await cookies()).get(LOCALE_COOKIE)?.value;
  if (cookieVal === "he" || cookieVal === "en") return cookieVal;

  const accept = (await headers()).get("accept-language") ?? "";
  // accept-language quality-list parsing is overkill here; we only support two
  // locales and just need to detect a Hebrew tag anywhere in the preference list.
  if (/(^|[,;\s])he(\b|-)/i.test(accept)) return "he";
  return DEFAULT_LOCALE;
}

export function isRtl(locale: Locale): boolean {
  return locale === "he";
}
```

```ts
"use server";

import { cookies } from "next/headers";
import { revalidatePath } from "next/cache";
import { LOCALE_COOKIE, SUPPORTED_LOCALES, type Locale } from "./locale";

export async function setLocale(locale: Locale): Promise<void> {
  if (!SUPPORTED_LOCALES.includes(locale)) return;

  (await cookies()).set(LOCALE_COOKIE, locale, {
    path: "/",
    maxAge: 60 * 60 * 24 * 365,
    sameSite: "lax",
  });
  revalidatePath("/", "layout");
}
```
